# 本命宝可梦（Birthday to Pokemon）· 微信小程序

根据生日日期映射到宝可梦全国图鉴编号（1–1025），并在小程序中展示本命宝可梦的名称与图标。

---

## 一、实现情况

### 1. 功能实现概览

| 需求 | 实现方式 |
|------|----------|
| 一年全部天数 → 1–1025 编号映射 | 先算「一年中的第 N 天」，再 `编号 = (N-1) % 1025 + 1`，输入生日即可得到唯一编号 |
| 按编号显示宝可梦名称 | 使用从神奇宝贝百科页面解析出的 1–1025 名称数据 `miniprogram/data/pokemon.json` |
| 按编号显示宝可梦图标 | 使用 PokeAPI 官方立绘，支持多图源切换（见下文图源列表） |
| 图源切换 | 首页「图片源」下拉列表切换 CDN，选择会持久化到本地；默认推荐 jsDelivr（国内更稳定） |
| 图片刷新 | 结果卡片下方提供「刷新图片」按钮，可重新加载当前图源地址（带时间戳防缓存） |
| 前后端形态 | 前端：微信小程序（选择日期、展示结果）；逻辑与数据均在小程序内，无独立后端服务 |

### 2. 映射规则说明

- **输入**：生日日期（年-月-日）。
- **步骤**：
  1. 计算该日期在当年是第几天（1 月 1 日为第 1 天，闰年 2 月 29 日为第 366 天）。
  2. 宝可梦编号 = `((天数 - 1) % 1025) + 1`，得到 1–1025 的整数。
- **示例**：2000-01-01 → 第 1 天 → 编号 1（妙蛙种子）；2000-12-31 → 第 366 天 → 编号 366（珍珠贝）。

### 3. 数据来源与脚本

- **名称**：来自 [神奇宝贝百科 - 宝可梦列表（按全国图鉴编号）](https://wiki.52poke.com/wiki/宝可梦列表（按全国图鉴编号）） 的本地保存 HTML。
- **生成方式**：运行 `scripts/extract-pokemon.js`，从该 HTML 中解析出每个编号第一次出现的名称（排除地区形态重复项），生成 `miniprogram/data/pokemon.json`。
- **图标**：通过 PokeAPI 的 official-artwork 按编号动态拼接 URL，无本地图片资源；图源在 `miniprogram/utils/util.js` 的 `IMAGE_SOURCE_LIST` 中配置。

#### 图源列表（支持切换）

| 图源名称 | 说明 | 域名 |
|----------|------|------|
| jsDelivr CDN（国内推荐） | 默认选项，国内访问更稳定 | `cdn.jsdelivr.net` |
| GitHub Raw | PokeAPI 官方仓库直链 | `raw.githubusercontent.com` |

切换图源后立即对当前结果生效，且下次打开小程序会保留上次选择。

#### 图片刷新

结果页展示本命宝可梦时，下方有 **「刷新图片」** 按钮。若当前图源偶发加载失败（如 `ERR_CONNECTION_RESET`），可先尝试切换图源，或点击刷新按钮重新请求图片。

## 二、微信开发者工具：测试与使用操作手册

### 1. 环境准备

- 安装 [微信开发者工具](https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html)。
- 用微信扫码登录（可选，不登录也可本地预览）。

### 2. 导入并打开项目

1. 打开微信开发者工具。
2. 选择 **「+」** 或 **「导入项目」**。
3. **目录**：选择本仓库根目录 `Birthday_to_Pokemon`（即包含 `project.config.json` 和 `miniprogram` 文件夹的目录）。
4. **AppID**：可选用 **「测试号」** 或 **「不使用 AppID」**（仅本地体验）。
5. 点击 **「导入」** 或 **「确定」**，等待编译完成。

#### 小程序模式 vs 小游戏模式（双入口兼容）

本项目同时提供**小程序**与**小游戏**两种入口，工具会根据当前编译类型加载对应入口：

| 模式 | 入口文件 | 说明 |
|------|----------|------|
| **小程序**（推荐） | `miniprogram/app.js` | 完整功能：生日选择、本命宝可梦展示、图源切换、图片刷新。 |
| **小游戏** | 根目录或 `miniprogram/` 下的 `game.js` | 完整功能：Canvas 界面下支持选择生日（年/月/日 ActionSheet）、查看本命宝可梦、图源切换、刷新图片，与小程序能力一致。 |

- 若导入后出现 **「game.js 未找到」**，说明当前被识别为小游戏。可重新导入并选择类型为 **「小程序」**，或直接编译运行：小游戏入口同样提供完整本命宝可梦功能。
- 根目录与 `miniprogram/` 下均保留 `game.js`、`game.json`；根目录 `game.js` 使用 `./miniprogram/` 下 util 与数据，`miniprogram/game.js` 使用相对路径 `./utils`、`./data`，便于工具无论从哪一级解析小游戏入口都能正确加载。

### 3. 图片域名配置（必做，否则图标不显示）

图标使用外链 CDN，需允许对应域名：

- **方式一（推荐本地调试）**  
  - 右上角 **「详情」** → **「本地设置」**  
  - 勾选 **「不校验合法域名、web-view（业务域名）、TLS 版本以及 HTTPS 证书」**  
  - 保存后重新编译，图标即可加载。

- **方式二（正式发布时）**  
  - 登录 [微信公众平台](https://mp.weixin.qq.com/) → 小程序  
  - **开发管理** → **开发设置** → **服务器域名**  
  - 在 **downloadFile 合法域名** 中增加当前图源所用域名，例如：  
    - `https://cdn.jsdelivr.net`（默认图源，国内推荐）  
    - `https://raw.githubusercontent.com`（GitHub Raw 图源）  
  - 保存后，在开发者工具中取消勾选「不校验合法域名」再测试。

### 4. 本地运行与测试

1. **编译**：工具会自动编译；若未自动编译，点击 **「编译」**。
2. **选择生日**：在模拟器中点击 **「生日」** 区域，选择日期（如 2000-01-01）。
3. **选择图源**（可选）：点击 **「图片源」** 下拉列表，可切换「jsDelivr CDN（国内推荐）」或「GitHub Raw」；国内网络不稳定时建议使用 jsDelivr。
4. **查看结果**：点击 **「查看我的本命宝可梦」**，应显示：
   - 全国图鉴编号（No.XXXX）；
   - 本命宝可梦名称；
   - 本命宝可梦立绘图片（若已按上一步配置域名）。
   - 若图片未加载出，可点击结果下方的 **「刷新图片」** 或切换图源后重新查看。
5. **真机预览**（可选）：点击 **「预览」**，用微信扫码在手机上查看。

### 5. 常见问题

| 现象 | 处理 |
|------|------|
| 图标不显示、报域名错误 | 按上文「图片域名配置」勾选不校验合法域名或配置对应图源域名（如 `cdn.jsdelivr.net`、`raw.githubusercontent.com`）。 |
| 部分图片加载失败（如 ERR_CONNECTION_RESET） | 多为网络波动或图源在国外被限速。在首页将 **「图片源」** 切到 **「jsDelivr CDN（国内推荐）」**，或点击结果页 **「刷新图片」** 重试。 |
| 选择日期后无反应 | 确认已点击 **「查看我的本命宝可梦」**；若仍无结果，查看调试器 Console 是否有报错。 |
| 缺少 `pokemon.json` | 在项目根目录执行 `node scripts/extract-pokemon.js`，确保存在 `宝可梦列表（按全国图鉴编号）- 神奇宝贝百科....html`。 |
| **game.js 未找到**（env 显示 mg） | 当前为小游戏模式。可重新导入并选择类型为 **「小程序」**；或直接运行，小游戏入口也支持完整功能（选择生日、查看本命宝可梦、图源、刷新）。 |
| 1970 年以前日期异常 | 部分环境下日期选择器对 1970 年前支持有限，建议用 1970 年及以后的日期测试。 |

### 6. 重新生成宝可梦数据（可选）

若更新了百科 HTML 或需重新生成 `pokemon.json`：

1. 将新的 HTML 放在项目根目录，或修改 `scripts/extract-pokemon.js` 中的 `htmlPath`。
2. 在项目根目录执行：  
   `node scripts/extract-pokemon.js`  
3. 确认 `miniprogram/data/pokemon.json` 已更新后，在开发者工具中重新编译。
